About Patrick Curl

Patrick Curl is a web developer, primarily focused on Laravel, Vue, Angular, and React.

He started making websites using geocities back in 1998. He then became a script ‘junky’, installing every script on hotscripts including such gems as PHPNuke. His first code was a custom widget for PHPNuke that had more spaghetti than all of Italy eats in a week.

He started using Wordpress in 2008 for personal projects and started helping others on their wordpress issues. In 2010 he began coding custom plugins - still using lots of spaghetti code.

In 2011 he began learning about OO programming concepts and how to use a REAL framework to write not-so-shitty-but-still-kind-of-shitty code. His first framework of choice was Ruby on Rails.

He spent 2 years on a number of small side projects that never were ‘good enough’ to go live with, but learned a shit-ton of do’s/don’ts about coding. In 2013 he was hired to build a book buy-back site for a company. He spent two weeks building it in Rails, but the client wanted to deploy to shared hosting. So, he decided to port everything to Laravel 4.2 that he’d done so far.

Porting to laravel 4.2 was a breeze, and so was everything else about laravel, so he said “F$ck ruby, I’m going to use Laravel for my projects from now on.” and by golly that’s what he did!

On the frontend he picked up Bootstrap back in 2013 as well, and mostly used jQuery but started playing around with Angular in some areas. In 2016 he began using Angular a lot more in production, as well as using it in an Ionic/Cordova app for a client. 2016 he also began using Vue in some areas, and started using that more and more.

2018 is a new year and he plans on trying to branch off in new directions, possible django/react. His goal is to eventually work with machine learning, so becoming proficient in python is a good step in that direction.

He’s currently for hire, hourly freelance rate is: $60.
